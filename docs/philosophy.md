# Philosophy

## Purpose

Rise Gate OS の目的は、業務を管理することだけではありません。

日々の案件、判断、文書、改善、そしてお客様との共有プロセスを蓄積し、会社の知識へ変え、AIが活用できる状態に育てることです。

このシステムは、Company OS の考え方を実際の業務へ落とし込む最初の Operating System です。

## Core Statement

改善を、文化に。

## Core Concepts

### Project

Project は改善を実行する単位です。

同時に、Project は社内だけの管理画面ではなく、お客様と一緒に改善を進める共有空間です。

顧客、タスク、工程、メンバー、文書、イベント、改善は Project を中心に整理します。

お客様も Project Members の一員として招待でき、権限に応じて進捗、工程、公開されたDocuments、Comments、Improvements、納品物を確認できます。

### Improvement

Improvement は会社の資産です。

初期実装では Project に紐付けますが、将来的には Client や Workspace の資産としても扱えるようにします。

改善は単なる履歴ではありません。

現状、理想、問題、仮説、行動、結果、影響、次の行動を残し、次の改善へつなげるための構造です。

### Documents

Documents は知識の器です。

議事録、仕様書、契約書、画像、PDF、作業報告、納品資料などを、単なる添付ファイルではなく業務文書として扱います。

### AI

AI は人を置き換えるものではありません。

蓄積された知識を探し、整理し、改善のきっかけを提案するパートナーです。

## Client Participation

Rise Gate OS では、お客様も Project の参加者として考えます。

Project は、ホームページ制作やシステム開発そのものではありません。

お客様と一緒に改善を進める場所であり、その成果物としてホームページ、システム、AI、マニュアル、運用改善などが生まれます。

一般的な制作現場では、メール、電話、チャット、ファイル共有、進捗管理が分散しがちです。

Rise Gate OS では Project ひとつに集約し、「今どうなっていますか？」という不安を減らし、改善をお客様と共有できる状態を目指します。

ただし、原価、利益、社内メモ、担当者評価、社内改善、内部タスク、内部コメントは社内限定として扱います。

## Operating Principles

- 完成品を最初から目指さない。
- 現場で使いながら育てる。
- 画面追加、機能追加、テーブル追加をしやすくする。
- 設計変更は判断の履歴を残す。
- コード、ドキュメント、改善履歴を同じ資産として扱う。
- ライズゲート専用に閉じず、将来ほかの中小企業にも展開できる設計にする。
- お客様と共有できる情報と、社内限定の情報を明確に分ける。

## For Members and Partners

Rise Gate OS は、ライズゲート本体だけでなく、弟子や独立したパートナーも使う前提です。

各メンバーは独立した Workspace を持ち、自分の顧客や案件を管理します。

必要に応じて、他のメンバーやお客様を Project に招待し、共同で案件を進めます。
