# Philosophy

## Purpose

Rise Gate OS の目的は、業務を管理することだけではありません。

日々の案件、判断、文書、改善、そしてお客様との共有プロセスを蓄積し、会社の知識へ変え、AIが活用できる状態に育てることです。

このシステムは、Company OS の考え方を実際の業務へ落とし込む最初の Operating System です。

Company OS は思想・概念です。会社の理念、判断基準、知識、改善をつなぎ、会社が進化し続けるための考え方です。

Rise Gate OS は実装・プロダクトです。Company OS の思想を、Project、Improvement、Documents、Dashboard、AI連携として実際の業務で動かすためのシステムです。

## Core Statement

改善を、文化に。

## Core Concepts

### Project

Project は改善を実行する単位です。

ただし、Project は主役ではありません。Project は改善を育てる器です。

同時に、Project は社内だけの管理画面ではなく、お客様と一緒に改善を進める共有空間です。

顧客、タスク、工程、メンバー、文書、イベント、改善は Project を中心に整理します。

お客様も Project Members の一員として招待でき、権限に応じて進捗、工程、公開されたDocuments、Comments、Improvements、納品物を確認できます。

### Roadmap

Roadmap は、Project が目指す未来へ向かうテーマです。

Roadmap は改善そのものではありません。

Roadmap は、複数の Improvement を生み出し、束ねる一段上の視点です。

たとえば `運用できるOS` という Roadmap があり、その中に `Evolution Dashboardを育てる`、`Taskを実行可能にする`、`権限を分かりやすくする` といった Improvement が生まれます。

Roadmap は最初から必須ではありません。改善が増え、Projectの未来への道筋を整理したくなったタイミングで育てます。

### Improvement

Improvement は会社の資産です。

初期実装では Project に紐付けますが、将来的には Client や Workspace の資産としても扱えるようにします。

改善は単なる履歴ではありません。

改善は管理するものではなく、育てるものです。

現状、理想、問題、仮説、行動、結果、影響、次の行動を残し、次の改善へつなげるための構造です。

### Documents

Documents は知識の器です。

議事録、仕様書、契約書、画像、PDF、作業報告、納品資料などを、単なる添付ファイルではなく業務文書として扱います。

### AI

AI は人を置き換えるものではありません。

蓄積された知識を探し、整理し、改善のきっかけを提案するパートナーです。

## 改善は育てるもの

Rise Gate OS では、改善を管理対象として扱いません。

改善は、生まれ、育ち、実行され、結果が残り、知識になり、次の改善を生み出します。

```txt
Improvement
  ↓
Action
  ↓
Result
  ↓
Knowledge
  ↓
Next Improvement
```

そのため、画面や機能の判断基準は「管理しやすいか」だけではありません。

利用者が改善を見つけやすく、育てやすく、結果を残しやすく、次の改善へ進みやすいかを重視します。

## Rise Gate OS自身も育つOS

Rise Gate OS は、完成してから使うシステムではありません。

Rise Gate OS 自身も、実際に使われながら Improvement を生み、実装され、結果を記録し、次の改善へつながる Operating System です。

つまり、Rise Gate OS は会社の改善を支援するだけでなく、自分自身も改善を続けながら育っていくプロダクトです。

設計、ドキュメント、UI、データ構造、運用ルールも改善対象として扱います。

## 改善が会社を進化させる

Company は Project を繰り返すだけで成長するのではありません。

Project の中で見つかった Improvement が、次の行動を生みます。

その行動は、小さな Task になることもあれば、新しい Project になることもあります。

```txt
Project
  ↓
Improvement
  ├── Task
  └── New Project
```

たとえば、ホームページ制作の中で「お問い合わせが増え、メール管理では限界」という Improvement が生まれた場合、それは単なる作業ではなく、CRMシステム開発という新しい Project につながる可能性があります。

採用情報を充実させたいという Improvement から採用サイト制作が生まれることもあります。

勤怠管理の改善から給与計算システムが生まれることもあります。

Rise Gate OS が管理するものは、Project の一覧ではありません。

Project で生まれた Improvement が、Task、Project、Document、Decision へ変わり、さらに次の Improvement を生む流れです。

この改善の連鎖こそが、Company OS における会社の進化です。

Phase 2-2では、この考え方の第一歩として、ImprovementからTaskやNew Projectを生み出し、その由来を辿れる構造を実装しました。

## Evolution Dashboard

Evolution Dashboard は、会社の未来へ進むためのホーム画面です。

過去を管理するための画面ではなく、昨日より会社が少し進化したことを感じ、今日育てる改善を見つけ、次に進みたくなる場所です。

Dashboardの主役は、不足、遅延、期限超過ではありません。

主役は、会社が少しずつ良くなっていることです。

Evolution Dashboard は、次の5つを大切にします。

- 進化を感じる。
- 育てる改善を見つける。
- 次の一歩を後押しする。
- 停滞を責めない。
- AIへの入口になる。

停滞している改善は、責めるために表示するのではありません。

「確認すると進められる改善」として扱い、前へ進む入口を示します。

将来的にはAIが Project、Improvement、Documents、Events、History を理解し、次に見るべき改善を提案します。

## Client Participation

Rise Gate OS では、お客様も Project の参加者として考えます。

Project は、ホームページ制作やシステム開発そのものではありません。

お客様と一緒に改善を進める場所であり、その成果物としてホームページ、システム、AI、マニュアル、運用改善などが生まれます。

一般的な制作現場では、メール、電話、チャット、ファイル共有、進捗管理が分散しがちです。

Rise Gate OS では Project ひとつに集約し、「今どうなっていますか？」という不安を減らし、改善をお客様と共有できる状態を目指します。

ただし、原価、利益、社内メモ、担当者評価、社内改善、内部タスク、内部コメントは社内限定として扱います。

## Operating Principles

- 完成品を最初から目指さない。
- Company OSは、最初から完成形を求めない。会社の成長に合わせて、必要な構造が自然に育っていくOperating Systemである。
- 改善を管理するのではなく、育てる。
- 現場で使いながら育てる。
- 画面追加、機能追加、テーブル追加をしやすくする。
- 設計変更は判断の履歴を残す。
- コード、ドキュメント、改善履歴を同じ資産として扱う。
- ライズゲート専用に閉じず、将来ほかの中小企業にも展開できる設計にする。
- お客様と共有できる情報と、社内限定の情報を明確に分ける。

## For Members and Partners

Rise Gate OS は、ライズゲート本体だけでなく、弟子や独立したパートナーも使う前提です。

各メンバーは独立した Workspace を持ち、自分の顧客や案件を管理します。

必要に応じて、他のメンバーやお客様を Project に招待し、共同で案件を進めます。
